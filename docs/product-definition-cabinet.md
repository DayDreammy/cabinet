# Product Definition Brief: Cabinet

**版本**：1.0

**角色定义**：首席案头研究员 (Chief Desk Researcher)

**核心价值**：以私人信源为基础的决策辅助与深度导读。

### 1. 愿景 (Vision)

在信息过载与AI幻觉的时代，为用户提供一个**绝对可信**、**过程透明**、**还原语境**的思考代理人。它不生产内容，它用你信任的智慧来审视你的难题。

### 2. 解决的核心问题 (The Problem)

- **搜索的局限**：关键词匹配无法理解复杂意图，且容易漏掉非显性的核心文章（Recall低）。
- **RAG的缺陷**：切片化导致逻辑断裂，丢失了专家论述的完整性（Context丢失）。
- **Chatbot的痛点**：黑盒等待让人焦虑，生成的总结往往不够精准，甚至编造事实（信任度低）。

### 3. 产品隐喻 (Core Metaphor)

- **不是**：帮你做饭的厨师（直接给加工好的结论）。
- **而是**：博物馆的**专业导览员**（带你找到原作，打上高光，让你自己看）。
- **消费单元**：“一顿饭的量”（按需提供 5-10 篇核心内容，而非海量链接）。

### 4. 核心架构逻辑 (Architectural Pillars)

#### A. 语义膨胀与多路召回 (Semantic Expansion)

- Agent 首先对用户模糊的意图进行“头脑风暴”，扩展出本质词、外延词、反义词。
- 确保即使原文没提关键词，但聊了相关概念的文章也能被捞起。

#### B. 浓度重排序 (Density Reranker)

- 拒绝单纯的向量相似度。
- 引入 LLM 裁判机制，计算文章对该主题的“含金量”（是整篇都在聊，还是顺带一提）。
- 输出分层菜单：**必读 (Core)** vs **选读 (Extended)**。

#### C. 以引用为中心 (Quote-Centric)

- **铁律**：严禁 AI 改写或总结。
- 输出必须是“原句摘录” + “推荐理由”。
- 点击摘录必须**100% 还原**到原文的具体段落。

### 5. 用户体验设计 (UX/UI Strategy)

#### 界面风格：The Transparent Lab (透明实验室)

- **参考对象**：Manus / Notion。
- **核心交互**：
    1. **立项**：用户输入困惑（Project Based），系统建立持久化任务。
    2. **透明工作流**：
        - 可视化展示 Agent 的思考（生成标签云）。
        - 可视化展示扫描过程（卡片飞入待读区）。
        - 可视化展示筛选逻辑（红叉剔除无关，绿勾保留核心）。
    3. **结果交付**：平滑过渡到“导读仪表盘”。

#### 核心视图：Split View (分栏导读)

- **左侧（导游）**：AI 生成的精选卡片流（原句+理由）。
- **右侧（原作）**：完整的、不可篡改的原文视图。
- **联动**：点击左侧卡片，右侧自动滚动并高亮对应段落。

### 6. 技术与数据策略 (Technical Strategy)

- **数据源**：用户指定的、高价值的私域信源（3-4k 篇）。
- **上下文策略**：放弃碎片化 RAG，针对筛选出的 Top 50 文章使用 **Long-Context Window (全文阅读)**，确保逻辑完整。
- **持久化**：所有搜索结果以“项目报告”形式永久存储，支持异步查看，无需重复计算。

---

### 架构师寄语 (Architect's Note)

**Cabinet** 的护城河不在于它搜得有多快，而在于它如何建立**信任**。

通过**过程的可视化**（Manus UI）和**结果的原文溯源**（Split View），我们把 AI 从一个“不可靠的聊天对象”变成了一个“严谨的研究助理”。这就是从 Tool 到 Solution 的跨越。
